# Column definitions

```tsx
interface StyleProps {
  width: number;
  minWidth: number;
  maxWidth: number;
  hidden: boolean;
  flex: number;
  menu?: boolean | Partial<MenuProps>;
  aggregationLevel?: number;
  aggregation?: AggregationType;
}
export interface MenuProps {
  column: boolean;
  filter: boolean;
  grid: boolean;
}
interface ColumnDef extends Partial<StyleProps> {
  headerName: string;
  field: string;
  children?: ColumnDef[];
  formatter?: (value: unknown) => string;

}
```

<Callout>
    The column definitions are used to define the columns of the grid. It is a recursive structure, so you can define [columns inside columns](nesting).
</Callout>

<Callout type="info">
  Check the example in the [Example](../example) section.
</Callout>

## Properties

### headerName

| Type | Required |
| ---- | -------- |
| `string` | `true` |

The name of the column.

### field

| Type | Required |
| ---- | -------- |
| `string` | `true` |

The field to be displayed in the column. Must match the field name in the data.

### children
 
| Type | Required |
| ---- | -------- |
| `ColumnDef[]` | `false` |

The children columns of the column.

### formatter

| Type | Required |
| ---- | -------- |
| `(value: unknown) => string` | `false` |

A function to format the value of the column.

### width

| Type | Required |
| ---- | -------- |
| `number` | `false` |

The width of the column.

### minWidth

| Type | Required |
| ---- | -------- |
| `number` | `false` |

The minimum width of the column.

### maxWidth

| Type | Required |
| ---- | -------- |
| `number` | `false` |

The maximum width of the column.

### hidden

| Type | Required |
| ---- | -------- |
| `boolean` | `false` |

Whether the column is hidden.

### flex

| Type | Required |
| ---- | -------- |
| `number` | `false` |

The flex value of the column. Follows the same rules as the flex property in CSS.

### menu

| Type | Required |
| ---- | -------- |
| `boolean` | `false` |

Whether to show the menu for the column. If `true`, it will show the default menu. If an object, it will show the menu with the properties specified in the object.

### aggregationLevel

| Type | Required |
| ---- | -------- |
| `number` | `false` |

If this level is set, data will be grouped by this column, on the specified level.

### aggregation

| Type | Required |
| ---- | -------- |
| `AggregationType` | `false` |

The type of aggregation to be used for this column. If not set, it will not show any data in the grouped row.

The aggregation type is one of the following:

- `sum`
- `avg`
- `min`
- `max`
- `count`

Check how it works in the [Row Grouping](../rows/grouping) section.
